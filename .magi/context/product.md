# Magi — プロダクト概要

## ミッション

3人のAIロール（PM・PD・Dev）が多角的に議論し、合意に基づいた高品質な成果物を生み出すシステム。
「1つのLLMに聞く」のではなく「3つの視点で議論してから決める」ことで、見落としを減らし、判断の経緯を記録する。

## 名前の由来

**Magi** — 東方三博士。異なる知恵を持つ3人の賢者が、それぞれの贈り物を持ち寄る。

## ターゲットユーザー

- **PM / PD**: Specモードで仕様策定。コードを書かなくても使える
- **エンジニア**: Buildモードで設計→実装→レビューを自律実行
- **チーム全体**: Web UIでリアルタイムに議論を閲覧・介入

## 2つのモード

### Specモード（仕様駆動）
課題を投入 → 3ロールが議論 → 要件定義 + 仕様書 + ADR + タスクリスト を生成。
実装は人間でもAIでも誰がやってもいい。

### Buildモード（実装駆動）
課題を投入 → 3ロールが議論 → 実装 → レビュー → 検証まで一気通貫。
全過程がGitに残る。

## 本質的な価値

| 従来のAIコーディング | Magi |
|---|---|
| 1つの視点で即座に回答 | 3つの視点で議論してから実行 |
| なぜそう作ったか不明 | 議論の経緯がGitに残る |
| 見落としは自己責任 | 3ロールが相互チェック |
| 設計判断の記録なし | 採用/却下理由が検索可能 |

## チーム共有

VS Codeのポート転送でWeb UIを共有。非エンジニアはブラウザだけで議論閲覧・介入が可能。
将来的には `magi publish` で静的サイト生成し、GitHub Pagesにデプロイ。
