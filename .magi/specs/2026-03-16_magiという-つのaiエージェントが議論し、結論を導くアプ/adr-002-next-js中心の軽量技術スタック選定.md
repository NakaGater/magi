# ADR-002: Next.js中心の軽量技術スタック選定

## ステータス
承認済み

## コンテキスト
2体AI議論システムの技術スタック選定において、開発効率、コスト効率、実現可能性のバランスを取る必要があった。検証段階では過度に複雑な技術構成は避けるべき状況だった。

## 決定
Next.js 14 + TypeScript + Tailwind CSS + OpenAI GPT-3.5-turbo + Vercel構成を採用する。

## 理由
- Next.js 14は開発効率が高く、API RoutesでBackend機能も統合可能
- OpenAI GPT-3.5-turboはコスト効率が良く予算内で検証可能
- Vercelは自動デプロイとAnalyticsを提供し運用コストを削減
- 1週間での実装が現実的

## 却下した代替案
- LangChain: 学習コストが高く検証段階では過剰
- AWS Lambda: 検証期間2週間では開発効率が劣る
- 複雑なDB設計: 単純テーブル構成で十分
- 高度な状態管理ライブラリ: MVPには不要

## 影響
- 開発期間1週間での実装が可能
- 検証期間総コスト3万円以下を実現
- 技術的負債は最小限に抑制
- Phase1への移行時の技術変更余地を保持
