# 要件定義: Web UIで過去の議論履歴が見れる機能

## 概要
Web UIで過去のMagi議論履歴を一覧表示・詳細参照可能にし、非エンジニアユーザーでも過去の議論結果を活用できるようにする。MVPの差別化要素である「議論の経緯が検索可能」を実現する。

## ユーザーストーリー
- US-1: 非エンジニアユーザーとして、Web UIで過去の議論一覧を時系列で確認したい / 受入基準: /historyページで議論一覧が日付順で表示される
- US-2: ユーザーとして、議論タイトルと結論サマリーから関連する過去議論を特定したい / 受入基準: タイトル、日付、summary、合意状況が一覧で表示される
- US-3: ユーザーとして、興味のある議論の詳細な経緯を確認したい / 受入基準: 詳細リンクから議論の全やりとりが閲覧可能

## 機能要件
- FR-1: `/api/history`エンドポイントで`.magi/discussions/`から議論履歴一覧を取得
- FR-2: Web UIの/historyページで議論履歴をカードリスト形式で表示
- FR-3: 各議論項目にタイトル、日付、summary（1行）、合意状況、参加ロールを表示
- FR-4: 詳細リンクから`/discussion/[id]`への遷移で既存のDiscussionLiveコンポーネントを活用した詳細表示
- FR-5: meta.yamlからHistoryItem型の構造化データを生成

## 非機能要件
- NFR-1: 実装期限は2日以内、複雑化時は即座に延期
- NFR-2: path.join()によるディレクトリトラバーサル対策を実装
- NFR-3: エラーファイルは無視して処理継続（全体停止を回避）
- NFR-4: 同期ファイル読み込みによる簡易実装（MVP段階では許容）
- NFR-5: 型安全性を保つHistoryItem型定義の実装

## スコープ外
- 全文検索機能
- 高度なフィルタリング機能
- CLI `magi history`コマンドの実装
- 非同期処理による最適化
- ページング機能

## 前提条件
- `.magi/discussions/`ディレクトリ構造が安定している
- meta.yamlフォーマットが既に定義されている
- 既存のDiscussionLiveコンポーネントが利用可能
- Web UIの基本フレームワークが実装済み
