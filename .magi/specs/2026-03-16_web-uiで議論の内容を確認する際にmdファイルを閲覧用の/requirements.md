# 要件定義: Web UIでのマークダウン表示改善

## 概要
Web UIで議論内容を表示する際に、マークダウン記法が生のまま表示されて見にくい問題を解決する。段階的アプローチによりMVP完成を阻害せずに最小限の視覚改善を実現する。

## ユーザーストーリー
- US-1: 非エンジニアのstakeholderとして、VS Codeポート転送での画面共有時に議論内容の構造を理解したい / 受入基準: 見出し、リスト、強調が視覚的に区別できること
- US-2: エンジニアとして、長い議論内容をWeb UIで閲覧する際に認知負荷を軽減したい / 受入基準: マークダウン記法と実際のコンテンツが視覚的に分離されていること
- US-3: スクリーンリーダーユーザーとして、議論内容を理解しやすい形式で読み上げられたい / 受入基準: マークダウン記法部分が薄色表示され、重要な情報が際立つこと

## 機能要件
- FR-1: マークダウン記法（##, **, -, `等）をtext-gray-500で薄色表示する
- FR-2: 既存のwhitespace-pre-wrap表示を維持したまま視覚的改善を行う
- FR-3: 正規表現ベースでCSS適用を行い、HTMLパースは使用しない
- FR-4: CSS適用失敗時は既存表示にフォールバックする

## 非機能要件
- NFR-1: 実装工数は2-3時間以内に完了すること
- NFR-2: 新規セキュリティリスクを導入しないこと
- NFR-3: バンドルサイズを増加させないこと
- NFR-4: 既存のロールカラー表示との整合性を維持すること
- NFR-5: StatementCardコンポーネントの既存Props変更を行わないこと

## スコープ外
- react-markdownライブラリの導入（v1.1で検討）
- 完全なHTML化されたマークダウンレンダリング
- マークダウンエディタ機能の実装
- 非エンジニア向けオンボーディング画面

## 前提条件
- 現在のwhitespace-pre-wrap表示ベースのアーキテクチャを維持する
- MVP完成（Buildモードコード生成、テスト追加、LSP統合）を最優先とする
- v1.1で完全なマークダウンレンダリングの投資判断を実際のユーザーフィードバックに基づいて行う
- 主要ユーザーは現在エンジニアであり、マークダウン記法の基本的理解がある
