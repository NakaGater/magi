# 要件定義: Web UIでのマークダウン議論表示機能

## 概要
Web UIで議論内容を閲覧する際に、マークダウン記法で書かれた議論内容を適切にレンダリングして表示し、ユーザビリティを向上させる機能を実装する。現在のプレーンテキスト表示では議論の構造が理解しづらく、特に非エンジニアユーザーの議論参加を阻害している。

## ユーザーストーリー
- US-1: 非エンジニアユーザーとして、Web UIで議論内容を閲覧する際に、見出しやリストが視覚的に区別されて表示されることで議論の構造を理解したい / 受入基準: 見出し、リスト、コードブロックが適切にレンダリングされ、議論の階層構造が視覚的に認識できること
- US-2: 議論参加者として、共有されたWeb UIでマークダウン記法を知らなくても議論内容を正確に理解したい / 受入基準: マークダウン記法がそのまま表示されず、意図された書式で表示されること
- US-3: エンジニアユーザーとして、コードブロックやインラインコードが通常テキストと区別されて表示されることで、技術的議論の内容を正確に把握したい / 受入基準: コードブロックが適切なスタイリングで表示され、インラインコードが識別可能であること

## 機能要件
- FR-1: StatementCardコンポーネントでマークダウンコンテンツをレンダリング表示する
- FR-2: 見出し（H1-H3）、リスト（順序付き/順序なし）、コードブロック、インラインコードの基本的なマークダウン要素をサポートする
- FR-3: 既存のロールカラー表示機能と統合し、発言者による色分けを維持する
- FR-4: XSS攻撃を防ぐため、許可された要素のみをレンダリングする
- FR-5: DiscussionLiveコンポーネントでのリアルタイム更新時にもマークダウンレンダリングを適用する

## 非機能要件
- NFR-1: react-markdownライブラリを使用し、バンドルサイズの増加を50KB以下に抑制する
- NFR-2: 既存のStatementCardスタイルとの視覚的整合性を保つ
- NFR-3: 長文議論の複数同時レンダリング時のブラウザパフォーマンスを維持する
- NFR-4: モバイル端末での表示パフォーマンスを劣化させない
- NFR-5: ダークテーマでのマークダウン要素の視認性を確保する
- NFR-6: スクリーンリーダー等のアクセシビリティツールとの互換性を維持する

## スコープ外
- 高度なマークダウン要素（テーブル、画像、リンク等）のサポート
- 詳細なスタイルカスタマイズやテーマ調整
- マークダウンエディタ機能の実装
- パフォーマンス最適化の詳細調整

## 前提条件
- 現在のWeb UIはwhitespace-pre-wrapでプレーンテキスト表示している
- 議論内容はLLMによって生成されるマークダウン形式のテキスト
- 既存のStatementCard、DiscussionLiveコンポーネントが実装済み
- CSS Modulesによるスタイル管理が可能
- MVP完成度90%の状態で、1-2日の開発工数が確保可能
