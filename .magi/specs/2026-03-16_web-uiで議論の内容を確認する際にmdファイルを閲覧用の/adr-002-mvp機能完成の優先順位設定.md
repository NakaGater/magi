# ADR-002: MVP機能完成の優先順位設定

## ステータス
承認済み

## コンテキスト
MVP90%完成時点で、ユーザビリティ改善（マークダウンレンダリング）と未実装の重要機能（Buildモードのコード生成、テスト機能、LSP統合）のどちらを優先すべきか判断が必要となった。リソースが限られる中で、ビジネス価値に直結する投資判断が求められている。

## 決定
未実装の重要機能の完成をMVPの最優先事項とし、ユーザビリティ改善は最小工数での実装に留める。完全な機能実装はv1.1で実際のユーザーフィードバックを基に投資判断する。

## 理由
1. **MVP定義の厳守**: 90%完成状態で新規機能追加は慎重に判断すべき
2. **ビジネス価値の明確性**: 未実装機能の方がMVPとしての基本価値により直結
3. **リスク対効果**: 現在の主要ユーザー（エンジニア）にとってマークダウン記法は許容範囲内
4. **データドリブン判断**: 実際のユーザーフィードバック収集後の投資判断がより合理的

## 却下した代替案
- **ユーザビリティ改善優先**: MVP完成が遅延し、基本機能の安定性検証が不十分になるリスク
- **並行開発**: リソース分散により両方の品質が低下する可能性
- **全機能延期**: チーム共有時の理解性問題は即座に対処が必要

## 影響
- MVP完成スケジュールの確実な達成
- 基本機能の安定性と品質の確保
- v1.1での機能拡張時により良いユーザーデータに基づく意思決定が可能
- 最小限のユーザビリティ改善により、完全延期よりもユーザー体験を向上
