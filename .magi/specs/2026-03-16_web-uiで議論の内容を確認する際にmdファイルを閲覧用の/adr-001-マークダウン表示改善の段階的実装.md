# ADR-001: マークダウン表示改善の段階的実装

## ステータス
承認済み

## コンテキスト
MVP90%完成時点で、Web UIでの議論表示においてマークダウン記法がそのまま表示される問題が発生。特にVS Codeポート転送でのWeb UI共有時に、非エンジニアstakeholderが議論内容を理解するのに認知負荷が高く、チーム共有という本プロダクトの核心価値に影響を与えている。またアクセシビリティの観点からも、スクリーンリーダーでの読み上げ時に理解困難な状況が生じている。

## 決定
段階的実装アプローチを採用する：
- Phase 1: CSS正規表現でマークダウン記法部分のみを薄色表示（2-3時間で実装）
- Phase 2: 完全なreact-markdownによるHTMLレンダリングはv1.1で検討

## 理由
1. **リスク最小化**: 新規セキュリティリスクを導入せず、既存のXSSリスクレベルを維持
2. **投資効率**: 最小工数（2-3時間）でユーザビリティ改善を実現
3. **MVP完成優先**: 未実装の重要機能（Buildモード、テスト、LSP統合）への影響を最小限に抑制
4. **即効性**: 視覚的ヒエラルキー改善により議論の構造把握が容易になる

## 却下した代替案
- **react-markdown即座導入**: バンドルサイズ増加（約50KB）、XSS対策の複雑化、実装工数1日程度が必要
- **現状維持**: 非エンジニアstakeholderの理解性問題、アクセシビリティ問題が未解決
- **完全なマークダウン機能の延期**: ユーザー体験の重要性を軽視し、チーム共有価値を損なう

## 影響
- 視覚的ノイズの軽減により、非エンジニアstakeholderの議論参加が促進される
- アクセシビリティの段階的改善が実現される
- MVP完成スケジュールへの影響が最小限に抑えられる
- v1.1での完全マークダウンレンダリング検討時の技術的基盤となる
