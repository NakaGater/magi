# 機能仕様書: Web UIマークダウン表示改善

## 概要
Web UIで議論内容を表示する際に、マークダウン記法が生のまま表示されて見にくい問題を解決する。段階的アプローチによりMVP完成を阻害せずに最小限の視覚改善を実現する。実装工数2-3時間以内で、CSS正規表現ベースによりマークダウン記法部分を薄色表示し、議論内容の可読性を向上させる。

## 画面仕様
### StatementCard表示改善
- 目的: マークダウン記法と実際のコンテンツを視覚的に分離し、議論内容の理解を容易にする
- 構成要素: 既存のStatementCardコンポーネントのテキスト表示部分
- 振る舞い: 
  - `##`, `###`等の見出し記法を`text-gray-500`で薄色表示
  - `**text**`の強調記法部分を`text-gray-500`で薄色表示
  - `-`, `*`, `+`のリスト記法を`text-gray-500`で薄色表示
  - `` ` ``のインラインコード記法を`text-gray-500`で薄色表示
  - 既存の`whitespace-pre-wrap`表示を維持
  - CSS適用失敗時は既存表示にフォールバック

### DiscussionLive表示改善
- 目的: リアルタイム議論での可読性向上
- 構成要素: DiscussionLiveコンポーネント内のStatementCard
- 振る舞い: StatementCardと同様のマークダウン記法薄色表示を適用

## API仕様
### 既存API維持
- メソッド: 変更なし
- リクエスト: 変更なし
- レスポンス: 変更なし
- 備考: サーバーサイドの変更は不要、クライアントサイドのCSS改善のみ

## データモデル
既存のデータモデルを維持。マークダウンテキストはそのまま保存し、表示時にのみCSS処理を適用。

## エラーハンドリング
### CSS適用失敗時の処理
- 正規表現処理エラー時: 既存の`whitespace-pre-wrap`表示にフォールバック
- スタイル競合発生時: 既存のロールカラー表示を優先
- ブラウザ互換性問題時: プレーンテキスト表示を維持

### デグレード対応
- JavaScript実行エラー: デフォルト表示を保持
- CSS読み込み失敗: 既存スタイルで表示継続
- パフォーマンス影響: 処理時間超過時は元表示にフォールバック

## セキュリティ考慮
### XSSリスク回避
- HTMLパースを行わない正規表現ベースの実装
- 既存のセキュリティレベルを維持
- ユーザー入力の直接的なDOM操作を回避

### 実装制約
- StatementCardの既存Props変更を禁止
- 新規セキュリティリスクの導入を禁止
- 外部ライブラリの追加を禁止（バンドルサイズ増加回避）

## 実装仕様
### CSS実装方針
```css
/* マークダウン記法の薄色表示 */
.statement-content {
  /* 見出し記法: ##, ### */
  /* 強調記法: **, __ */
  /* リスト記法: -, *, + */
  /* コード記法: ` */
}
```

### 正規表現パターン
- 見出し: `/^#{1,6}\s/gm`
- 強調: `/\*\*(.*?)\*\*/g`, `/__(.*?)__/g`
- リスト: `/^[\s]*[-*+]\s/gm`
- インラインコード: `/`([^`]+)`/g`
