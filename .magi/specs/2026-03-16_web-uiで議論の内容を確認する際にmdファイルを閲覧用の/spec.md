# 機能仕様書: Web UIでのマークダウン議論表示機能

## 概要
Web UIで議論内容を閲覧する際に、マークダウン記法で書かれた議論内容を適切にレンダリングして表示し、ユーザビリティを向上させる機能の仕様書。議論の結果、MVP完成を最優先とし、CSS微調整による最小限の改善を実装し、完全なマークダウンレンダリング機能はv1.1で検討することが決定された。

## 画面仕様
### StatementCard（議論表示カード）
- 目的: 議論内容をマークダウン記法から視覚的に理解しやすい形式で表示
- 構成要素: 
  - 発言者ロール表示（既存機能維持）
  - 議論内容表示エリア（マークダウン記法の視覚的軽減）
  - タイムスタンプ（既存機能維持）
- 振る舞い: 
  - マークダウン記法部分（##, **, -, ` 等）を薄いグレー色で表示
  - 既存のwhitespace-pre-wrap表示を維持
  - ロールによる色分け表示を維持

### DiscussionLive（リアルタイム議論表示）
- 目的: リアルタイム更新される議論でも視覚的改善を適用
- 構成要素: 複数のStatementCardの集合
- 振る舞い: 新規追加される議論にも同様のCSS改善を自動適用

## API仕様
### 既存APIの変更なし
- メソッド: 変更なし
- リクエスト: 変更なし
- レスポンス: 変更なし
- 備考: フロントエンド表示改善のみのため、API仕様に変更は発生しない

## データモデル
既存のデータモデルに変更なし。議論内容は引き続きマークダウン形式のテキストとして保存される。

## エラーハンドリング
- CSS適用失敗時: デフォルトの既存表示（whitespace-pre-wrap）にフォールバック
- 長文議論での表示遅延: 現在の表示方式を維持するため新規エラーケースは発生しない

## セキュリティ考慮
- XSSリスク: HTMLパースを行わない正規表現ベースのCSS適用のため、新規セキュリティリスクは導入されない
- 既存のLLM生成コンテンツ表示と同等のセキュリティレベルを維持

## 実装詳細
### Phase 1: CSS微調整（MVP向け - 2-3時間）
```css
.statement-content {
  /* マークダウン記法部分の視覚的軽減 */
  /* ##, **, -, ` 等をtext-gray-500で表示 */
}
```

### Phase 2: 完全マークダウンレンダリング（v1.1向け）
- react-markdownライブラリの導入
- rehype-sanitizeによるXSS対策
- パフォーマンス最適化（メモ化）
- バンドルサイズ制約（50KB以下）
- 非エンジニアユーザーのユーザビリティテスト結果に基づく実装判断

## 制約事項
- MVP期間中は完全なマークダウンレンダリングは実装しない
- 既存の技術負債（テスト0件、Buildモード未完成、LSP統合未着手）の解決を最優先とする
- 現在の主要ユーザー（エンジニア）向けの最小限改善に留める
