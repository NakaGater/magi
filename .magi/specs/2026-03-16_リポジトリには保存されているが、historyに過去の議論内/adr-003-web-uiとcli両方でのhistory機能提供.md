# ADR-003: Web UIとCLI両方でのHistory機能提供

## ステータス
承認済み

## コンテキスト
History機能をWeb UIのみで提供するかCLIも含めるかの選択において、PMが効率性からCLI後回しを提案したが、PDがユーザー格差の懸念を示し、Devがロジック再利用の容易性を指摘した。

## 決定
Web UIとCLI両方でHistory機能を提供し、同一のロジックを再利用する実装とする。

## 理由
- ユーザー間での機能格差を回避できる
- CLI派ユーザーの過去議論確認ニーズに対応できる
- 同一ロジック再利用により追加実装コストを最小化できる
- ユーザーの混乱を防げる

## 却下した代替案
- Web UIのみ実装：CLI派ユーザーが機能を利用できず格差が生まれる
- CLIスタブのまま放置：ユーザーが過去議論を確認できない状況が継続する

## 影響
全ユーザーが一貫したHistory機能を利用でき、実装ロジックの再利用により開発効率も維持できる。
